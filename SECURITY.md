# セキュリティポリシー

WorldRec は VRChat のログ・認証情報や Gemini API キーといった機微な情報を扱うため、
セキュリティを重視しています。脆弱性の報告に協力いただける場合は以下の方針に従ってください。

## サポート対象バージョン

本プロジェクトは現在ベータ版（0.1.x）です。セキュリティ修正は最新リリースに対して提供します。

| バージョン | サポート |
|-----------|:-------:|
| 0.1.x（最新ベータ） | ✅ |
| それ以前 | ❌ |

## 脆弱性の報告

脆弱性を発見した場合は、**公開 Issue を作成しないでください**。悪用を防ぐため、
GitHub の [Security Advisories](../../security/advisories/new)（Private vulnerability reporting）から
**非公開で**報告してください。

報告には可能な範囲で以下を含めてください。

- 脆弱性の概要と想定される影響
- 再現手順（PoC があれば）
- 対象バージョンと動作環境（OS など）
- 考えられる緩和策や修正案（あれば）

### 対応の流れ

1. 報告の受領を確認します。
2. 再現・影響範囲を調査します。
3. 修正を作成し、最新ベータへ反映します。
4. 修正リリース後、報告者の希望に応じて謝辞を記載します。

責任ある開示にご協力いただいた報告者に感謝します。修正がリリースされるまでは、
脆弱性の詳細を公開しないようお願いします。

## データの取り扱いについて

WorldRec は以下の情報を扱います。利用者・報告者はこの前提を理解してください。

- **VRChat のログファイル**を読み取り、訪問履歴を SQLite（`%LOCALAPPDATA%\WorldRec\worldrec.db`）に
  **ローカル保存**します。履歴が外部サーバーへ自動送信されることはありません。
- **VRChat / Gemini の認証情報（API キー）は OS のキーリング（Windows 資格情報マネージャー）に保存**し、
  平文で設定ファイルやログに残しません。
- **AI 推薦機能を利用したときのみ**、ワールド情報などが **Gemini API（Google）へ送信**されます。
  この場合の取り扱いは Google のプライバシーポリシーに従います。

## 報告・ログ共有時の注意

不具合報告やログを共有する際は、**API キー・認証トークン・セッション情報・個人を特定しうる情報**が
含まれていないことを必ず確認してください。ログを添付する前に、該当箇所をマスクすることを推奨します。
